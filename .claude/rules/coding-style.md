# 编码风格规则

## 强制规则

### 1. 不可变性
- 始终创建新对象，不改变现有对象
- 使用 const 声明，避免 let
- 数组使用 map/filter/reduce 而非 push/splice

### 2. 文件组织
- 目标: 200-400 行/文件
- 绝对上限: 800 行/文件
- 按功能分组代码

### 3. 函数规范
- 长度 < 50 行
- 嵌套 ≤ 4 层
- 参数 ≤ 4 个
- 单一职责

### 4. 命名规范
- 变量/函数: 有意义的描述性名称
- 布尔值: is/has/can/should 前缀
- 常量: UPPER_SNAKE_CASE
- 类/接口: PascalCase

### 5. 错误处理
- 每个层级处理错误
- 提供有意义的错误消息
- 绝不默默吞掉错误
- UI 层显示用户友好消息

### 6. 注释
- 代码应该自解释
- 只在复杂逻辑处添加注释
- 不保留注释掉的代码
- TODO 标注负责人和截止日期
