# 安全规则

## 强制检查（提交前）

### 密钥管理
- [ ] 无硬编码的 API 密钥
- [ ] 无硬编码的密码
- [ ] 无硬编码的令牌
- [ ] 使用环境变量管理敏感配置

### 输入验证
- [ ] 所有用户输入已验证
- [ ] 参数化查询（防 SQL 注入）
- [ ] HTML 输出已转义（防 XSS）
- [ ] 文件上传已验证类型和大小

### 认证授权
- [ ] 认证逻辑正确
- [ ] 授权检查完整
- [ ] CSRF 保护已启用
- [ ] 会话管理安全

### API 安全
- [ ] 端点有速率限制
- [ ] 响应不暴露敏感信息
- [ ] 错误消息不泄露内部信息
- [ ] CORS 配置正确

## 安全事件响应

发现漏洞时：
1. 立即停止当前工作
2. 评估漏洞严重性
3. 优先修复关键漏洞
4. 替换已泄露的凭证
5. 对代码库做全面安全审计
