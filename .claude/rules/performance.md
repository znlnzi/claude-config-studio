# 性能优化规则

## 模型选择策略

**Haiku**（90% 的 Sonnet 能力，3x 成本节省）:
- 轻量级 Agent，频繁调用场景
- 结对编程和代码生成
- 多 Agent 系统中的 Worker

**Sonnet**（最佳编码模型）:
- 主要开发工作
- 编排多 Agent 工作流
- 复杂编码任务

**Opus**（最深推理）:
- 复杂架构决策
- 最大化推理需求
- 研究和分析任务

## 上下文窗口管理

避免在上下文窗口最后 20% 执行：
- 大规模重构
- 跨多文件的功能实现
- 复杂交互的调试

低上下文敏感度任务：
- 单文件编辑
- 独立工具函数创建
- 文档更新
- 简单 Bug 修复

## 构建故障排除

构建失败时：
1. 使用 build-error-resolver Agent
2. 分析错误消息
3. 增量修复
4. 每次修复后验证
