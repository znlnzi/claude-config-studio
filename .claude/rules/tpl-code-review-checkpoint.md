# 项目规范

## 检查点评估流程
在每个重要修改节点执行评估：

### 检查点 1: 实现前
- [ ] 理解了需求
- [ ] 研究了相似的现有实现
- [ ] 确定了实现方案

### 检查点 2: 实现后
- [ ] 代码可以编译通过
- [ ] 没有引入新的 lint 警告
- [ ] 核心逻辑有测试覆盖

### 检查点 3: 提交前
- [ ] 所有测试通过
- [ ] 没有遗留的 TODO 或 FIXME
- [ ] 代码可读性良好
- [ ] 没有冗余代码

## 冗余代码清理
定期检查并清理：
- 未使用的导入
- 未使用的变量和函数
- 重复的代码块
- 过时的注释
