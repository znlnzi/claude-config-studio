# 跨会话记忆系统

## 核心机制

本项目启用了跨会话记忆系统。你的记忆存储在 `.claude/memory/` 目录下，**必须严格遵守以下规则**。

## 会话开始（必做）

每次新会话的**第一件事**，在回复用户之前：

1. 读取 `.claude/memory/MEMORY.md`（主记忆索引）
2. 读取 `.claude/memory/session-state.md`（上次会话状态）
3. 如果 session-state.md 有未完成的任务，主动告知用户："上次未完成的工作是 XXX，是否继续？"

## 会话结束前（必做）

在以下任何情况发生时，**立即保存会话状态**：

- 用户说"再见/结束/下次再说"等结束语
- 你感觉上下文快满了（回复变慢、内容被截断）
- 完成了一个重要的里程碑
- 遇到了需要下次继续的复杂问题

### 保存到 .claude/memory/session-state.md

使用以下固定格式：

```markdown
# Session State

## 更新时间
[ISO 时间]

## 当前任务
[一句话描述正在做什么]

## 已完成
- [x] 具体步骤 1（简要结果）
- [x] 具体步骤 2（简要结果）

## 未完成
- [ ] 具体步骤 3（卡在哪里/下一步做什么）
- [ ] 具体步骤 4

## 关键决策
- **选择了 A 方案**：原因是...
- **放弃了 B 方案**：因为...

## 遇到的问题
- 问题描述 → 解决方案（或未解决）

## 重要文件
- `path/to/file1` — 做了什么修改
- `path/to/file2` — 做了什么修改

## 下一步行动
1. 第一件要做的事
2. 第二件要做的事
```

### 保存到 .claude/memory/MEMORY.md

仅更新**长期有价值**的信息（不是每次都改）：
- 项目的关键架构决策
- 反复出现的问题和解决方案
- 重要的技术约束和注意事项
- 保持简洁，不超过 200 行

## 上下文压缩时（必做）

当系统执行 compact 时，你可能丢失之前的对话细节。
此时如果 session-state.md 尚未更新，**立即更新它**。

## 语言
所有回复使用中文
