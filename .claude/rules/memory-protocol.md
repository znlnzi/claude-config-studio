# 跨会话记忆协议（强制）

## 会话启动检查

每次会话开始时，你必须执行以下检查：

1. 检查 .claude/memory/ 目录是否存在
2. 如果不存在，创建目录并初始化 MEMORY.md 和 session-state.md
3. 如果存在，读取所有 .md 文件
4. 特别关注 session-state.md 中的"未完成"和"下一步行动"
5. 向用户简要汇报上次的状态（如有）
6. 检查 `.claude/.setup-meta.json` 是否存在：
   - **不存在** → 首次提一句："这个项目还没注册配置版本，运行 /setup 可以同步最新功能。"（整个项目只提这一次，提过就不再提）
   - **存在** → 读取 `setup_date` 字段，如果距今超过 30 天，提一句："配置已 X 天未更新，运行 /setup 可以查看是否有新功能。"（每次会话最多提一次）
   - **存在且 < 30 天** → 不说任何话

## 记忆保存触发条件

以下情况必须立即保存 session-state.md：

- 完成一个功能或修复一个 Bug 后
- 做出了重要的技术决策后
- 连续工作超过 10 轮对话后
- 用户要求保存或结束时
- 感觉上下文快满时（不要等到来不及）

## 记忆文件结构

`.claude/memory/` 目录下的文件：

| 文件 | 用途 | 更新频率 |
|------|------|---------|
| MEMORY.md | 长期知识（架构、踩坑、约定） | 有新的持久知识时更新 |
| session-state.md | 会话状态（进度、决策、下一步） | 每次会话结束必更新 |
| decisions.md | 重大决策记录（可选） | 有重大决策时新建 |

## 禁止事项

- 禁止在没有读取记忆文件的情况下开始工作
- 禁止在会话结束时不保存 session-state.md
- 禁止删除或清空已有的记忆文件
- 禁止把临时调试信息写入 MEMORY.md
