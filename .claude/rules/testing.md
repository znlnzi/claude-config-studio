# 测试规则

## 覆盖率要求
- 最低测试覆盖率: 80%
- 关键路径: 100%（认证、支付、安全）

## TDD 流程（强制）

1. **RED** - 编写失败测试
2. **验证** - 确认测试失败
3. **GREEN** - 实现最小代码
4. **验证** - 确认测试通过
5. **REFACTOR** - 重构代码
6. **验证** - 确认覆盖率达标

## 测试类型要求

### 单元测试
- 隔离单个函数
- 模拟所有外部依赖
- 执行速度 < 50ms
- 无共享状态

### 集成测试
- 验证 API 端点
- 测试数据库交互
- 验证服务间通信

### E2E 测试
- 覆盖关键用户流程
- 使用 Playwright
- 测试独立运行

## 测试失败处理

测试不通过时：
1. 分析失败原因
2. 检查测试隔离性
3. 验证模拟实现
4. 修正实现（不是修改测试，除非测试有误）
