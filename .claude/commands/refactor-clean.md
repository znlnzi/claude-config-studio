# /refactor-clean - 死代码清理

安全地检测和移除未使用的代码。

## 工作流

### 1. 检测
运行静态分析工具检测：
- 未使用的导入
- 未调用的函数
- 未引用的变量
- 注释掉的代码块
- 未使用的依赖包

### 2. 分类
按风险级别分类：
- **SAFE**: 可安全删除（私有未使用代码）
- **CAUTION**: 需要确认（公共 API、动态引用）
- **DANGER**: 高风险（配置、入口文件）

### 3. 清理
按风险从低到高逐步清理：
1. 运行测试 → 确认通过
2. 删除 SAFE 类代码
3. 运行测试 → 确认通过
4. 处理 CAUTION 类（逐个确认）
5. 运行测试 → 确认通过

### 4. 验证
- 完整测试套件通过
- 构建成功
- 无新的 lint 警告

## 关键规则
**在运行测试之前，绝不删除代码！**
