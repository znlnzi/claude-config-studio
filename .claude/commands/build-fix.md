# /build-fix - 修复构建错误

系统化地分析和修复构建错误。

## 工作流

1. **收集错误** - 运行构建，捕获所有错误信息
2. **分类排序** - 按根源错误和级联错误分类
3. **逐一修复** - 从根源错误开始，每次修复一个
4. **验证修复** - 每次修复后重新构建
5. **完成确认** - 构建完全通过

## 修复策略

### 编译错误
- 检查类型定义和接口
- 确认导入路径正确
- 验证函数签名匹配

### 依赖错误
- 运行 npm install / go mod tidy
- 检查版本兼容性
- 解决依赖冲突

### 配置错误
- 检查构建工具配置文件
- 验证环境变量
- 确认路径设置

## 关键规则
- 每次只修复一个错误
- 修复后立即验证
- 最多尝试 3 次，失败则换策略
- 记录所有失败的尝试
