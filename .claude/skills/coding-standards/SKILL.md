# 编码标准技能

## 通用原则

### 不可变性
- 始终创建新对象，不改变现有对象
- 使用 const/final/val 声明不变量
- 防止隐藏的副作用

### 文件组织
- 每个文件 200-400 行，最大 800 行
- 按功能分组，不按文件类型
- 一个文件一个主要职责

### 函数设计
- 每个函数 < 50 行
- 单一职责
- 有意义的命名（动词+名词）
- 参数不超过 4 个

### 错误处理
- 每个层级实施错误管理
- UI 层提供用户友好消息
- 服务端记录详细上下文
- 绝不默默吞掉错误

### 输入验证
- 在系统入口点验证所有外部数据
- 使用基于模式的验证（Zod、Pydantic）
- 快速失败并提供描述性反馈

## 完成前检查清单

- [ ] 代码可读性和命名质量
- [ ] 函数 < 50 行
- [ ] 文件 < 800 行
- [ ] 嵌套 ≤ 4 层
- [ ] 健壮的错误处理
- [ ] 无硬编码配置值
- [ ] 不可变模式一致应用
