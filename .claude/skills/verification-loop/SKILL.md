# 验证循环技能

## 验证触发点

### 编码中
- 每完成一个函数 → 运行相关测试
- 每修改一个文件 → 类型检查

### 功能完成后
- 运行完整测试套件
- 检查覆盖率
- 运行 lint

### 提交前
- 构建验证
- 全量测试
- 安全扫描
- 调试代码审计

## 验证步骤

### 快速验证（编码中）
1. 类型检查通过
2. 相关测试通过
3. 无 lint 错误

### 完整验证（提交前）
1. 构建成功
2. 类型检查通过
3. Lint 通过
4. 全量测试通过
5. 覆盖率达标
6. 无调试语句
7. 无安全问题

## 失败处理

验证失败时：
1. 定位失败的步骤
2. 分析根本原因
3. 修复问题
4. 重新运行完整验证
5. 确认所有步骤通过

## 报告格式

验证完成后输出：
- 通过/失败状态
- 每个步骤的结果
- 失败项的详细信息
- 修复建议
